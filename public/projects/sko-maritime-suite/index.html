📌 Project Overview#

During my tenure at SKO Systems, I took end-to-end ownership of the technical, functional, and user-facing documentation assets for their comprehensive suite of maritime enterprise applications. These products streamline high-stakes cruise operations, guest logistics, biometric security, and international safety compliance.

  • Role: Senior Technical Writer
  • Target Audience: Field crew members, maritime port authorities, database administrators, and enterprise software integrators.
  • Core Methodologies: Microsoft Technical Publications (MSTP) standards, single-source requirements tracking, and Agile lifecycle documentation.

🛠️ The Challenge#

The maritime and cruise software ecosystem demands flawless precision; field crew members operate under strict, time-critical constraints during vessel embarkation, port clearance, and safety drills. Legacy documentation lacked structural consistency, creating bottlenecks during product deployment and lengthening onboarding cycles for crew training.

💡 The Strategy & Execution#

1. Standardizing Governance via MSTP#

I applied specialized structural writing principles and strictly enforced Microsoft Manual of Style (MSTP) rules. This involved translating dense engineering configuration scripts into clean, bite-sized instruction sets that could be rapidly executed by crew members on deck.

2. Business & Engineering Artifact Alignment#

Operating directly within Agile/Scrum sprints via Azure DevOps, I bridge-mapped product goals into technical realities by gathering requirements from stakeholders and authoring core project blueprints:

  • Business Requirement Documents (BRD) to capture high-level commercial logic.
  • Functional Requirement Documents (FRD) to map out detailed operational workflows.
  • Software Requirement Documents (SRD) to establish precise technical thresholds for developer references.

🌐 The Documented Ecosystem (Live Product Pages)#

I architected and authored the baseline layout structures, feature definitions, and operational workflows for the following 6 core modules:

  • mBark#

    • Focus: Next-generation mobile embarkation frameworks.
    • Documentation Core: Accelerated guest check-in workflows, wireless synchronization profiles, and offline fallback procedures for mobile port crew members.

  • Gangway Suite#

    • Focus: Real-time gangway security control systems.
    • Documentation Core: Technical deployment runbooks for monitoring crew, visitor, and passenger movements, alongside security hardware integration configurations.

  • SKO Face#

    • Focus: Biometric facial recognition identity verification layers.
    • Documentation Core: API reference setups for matching algorithms, camera sensor hardware alignment guides, and high-security compliance configurations.

  • sMuster#

    • Focus: Automated muster assembly station logging and emergency management.
    • Documentation Core: Mission-critical life safety instruction flows, emergency drill step-by-steps, and international maritime safety standard compliance mapping.

  • dBark Cruise#

    • Focus: Intelligent digital debarkation management.
    • Documentation Core: Complex logic parameters for grouping color-coded exit sequencing, luggage tracking handoffs, and port customs processing run-sheets.

  • sSManifest#

    • Focus: Electronic border-agency passenger manifest reporting.
    • Documentation Core: Data dictionary schema documentation for secure XML/JSON manifest outputs required by global customs authorities and harbor masters.

📈 The Business Impact#

  • Operational Optimization: Simplifying complex configuration scripts drastically reduced manual deployment errors by field installation crews.
  • Engineering Alignment: Maintaining current, robust SRDs and FRDs inside Azure DevOps effectively eliminated gaps between business requirements and QA testing targets.
  • Single Source of Truth: Unifying the voice across all 6 applications under MSTP guidelines built instant product cohesion, enhancing cross-training efficiency for users navigating multiple modules.
